# AT 3.0 違規統計功能說明

## 功能概述

在數字選擇器（A-K）的右側新增了**三個即時違規統計器**，用於監控牌靴生成和編輯過程中的違規情況。

---

## 三個違規統計器

### 1. 訊號牌違規
**顯示位置**: 最左側  
**統計內容**:
- **S局違規**: 有訊號牌的局，但下一局沒有開莊

**違規條件**:
```
當前局有訊號牌 → 下一局必須是「莊」
```

### 2. 連續5局4張
**顯示位置**: 中間  
**統計內容**: 檢測是否存在連續 5 局或以上都是 4 張牌的情況

**違規條件**:
```
連續 5 局或更多局都是 4 張牌（不補牌）
```

### 3. 連續7局莊閒
**顯示位置**: 最右側  
**統計內容**: 檢測是否存在連續 7 局或以上都開莊或都開閒的情況（和局不計入）

**違規條件**:
```
連續 7 局或更多局都是「莊」
或
連續 7 局或更多局都是「閒」
注意：和局會被跳過，不打斷計數也不增加計數
```

---

## 視覺效果

### 無違規狀態（綠色）
- 數字顯示為**綠色**（抹茶色）
- 邊框保持原色
- 背景為淺色

### 有違規狀態（紅色 + 動畫）
- 數字顯示為**紅色**（朱色）
- 邊框變為紅色
- 背景帶有淡紅色
- 數字會有**呼吸動畫效果**，引起注意

---

## 自動更新時機

違規統計會在以下操作後**自動更新**：

1. **生成牌靴** - 點擊 GO 按鈕後
2. **套用編輯** - 點擊「套用」按鈕後
3. **換牌操作** - 點擊「換牌」按鈕後
4. **換局操作** - 點擊「換局」按鈕後
5. **換色操作** - 點擊「換色」按鈕後
6. **清空牌靴** - 點擊「清空」按鈕後（所有違規數歸零）

---

## 手動編輯後的扣除機制

當你手動修改牌靴後（例如交換牌、修改局數結果等），系統會：

1. **重新計算所有違規**
2. **自動扣除已修正的違規**
3. **即時更新顯示數字**

例如：
- 原本有 3 個訊號牌違規
- 你手動修改了其中 2 個局的結果
- 系統重新計算後，違規數會變成 1

## 技術實現

### 核心函數

#### `calculateViolationStats(rounds)`
計算當前牌靴的所有違規統計
```javascript
return {
    signalViolations: 數量,      // 訊號牌違規數
    fourCardViolations: 數量,    // 連續4張牌違規數
    streakViolations: 數量,      // 連續7局莊閒違規數
    fourCardBlocks: [],          // 詳細區塊資訊
    streakBlocks: []             // 詳細區塊資訊
}
```

#### `updateViolationUI(stats)`
更新 UI 顯示，包括：
- 更新數字
- 切換顏色樣式
- 添加/移除動畫效果

#### `refreshViolationStats()`
從 `currentRounds` 重新計算並更新所有違規統計

### 整合點

- 在 `signals.js` 中添加了違規計算邏輯
- 在 `signals_ui.js` 中添加了 UI 整合和事件監聽
- 在 `index.html` 中添加了 HTML 結構和 CSS 樣式

## 使用建議

1. **生成牌靴前**: 確認訊號牌設定正確
2. **生成牌靴後**: 檢查違規統計，如有違規數字顯示，需要處理
3. **手動修正**: 使用換牌、換局等功能修正違規
4. **再次檢查**: 修正後違規數應該減少或歸零
5. **綠色為目標**: 所有違規統計都顯示 0（綠色）才是理想狀態

## 樣式說明

### CSS 類別

- `.violation-stats`: 違規統計區域容器
- `.violation-card`: 單個違規統計卡片
- `.violation-label`: 違規類型標籤
- `.violation-count`: 違規數字顯示
- `.no-violation`: 無違規狀態（綠色）
- `.has-violation`: 有違規狀態（紅色+動畫）

### 動畫效果

```css
@keyframes pulseViolation {
    0%, 100% { transform: scale(1); opacity: 1; }
    50% { transform: scale(1.05); opacity: 0.85; }
}
```

## 常見問題

### Q: 為什麼我修改了牌靴，違規數沒有變化？
A: 請確認你點擊了「套用」按鈕。只有套用後，系統才會重新計算違規統計。

### Q: 違規數顯示 0，但我看到有問題？
A: 違規統計只檢查三種特定違規。其他類型的問題（如莊6點等）不在統計範圍內。

### Q: 可以點擊違規統計查看詳細資訊嗎？
A: 目前版本只顯示數量。詳細的違規局數資訊可以在日誌區查看。

### Q: 手動編輯後，違規數如何扣除？
A: 系統會完整重新計算整個牌靴，所有已修正的違規會自動從統計中移除。
